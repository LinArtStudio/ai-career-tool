## 一、AI产品经理面试高频题（含参考答案）

### Q1：你如何理解AI产品经理和传统产品经理的区别？

**参考答案**：
AI产品经理需要额外关注三个维度：
1. **数据思维**：AI产品的迭代逻辑是"数据→模型→体验"的闭环，不是"需求→设计→开发"
2. **AI能力边界**：知道AI能做什么、不能做什么，避免设计出"伪AI需求"
3. **评估指标特殊性**：传统PM看DAU/GMV，AI PM还要看准确率、召回率、用户满意度等

### Q2：如何评估一个AI功能是否值得做？

**参考答案**：我会用三个维度评估：
1. **用户价值**：这个痛点是否足够痛？用户是否愿意为此付费？
2. **技术可行性**：当前AI能力是否能做出用户可接受的效果？准确率阈值是多少？
3. **商业回报**：开发成本vs预期收益，ROI是否合理？

### Q3：如果AI功能的准确率只有70%，你会怎么做？

**参考答案**：
- 首先看**场景容忍度**：简历诊断70%准确率可能够用，但自动填网申70%就不行
- 然后考虑**兜底策略**：AI不确定时提示"建议人工复核"，而不是硬给错误结果
- 最后做**渐进优化**：收集bad case → 分析错误模式 → 优化prompt/微调模型

### Q4：你认为AI求职工具最大的挑战是什么？

**参考答案**：
1. **用户信任**：AI给出的建议是否可信？需要建立信任机制（如解释推理过程）
2. **效果量化**：如何证明AI建议真的帮用户拿到了面试/offer？需要追踪闭环数据
3. **成本控制**：大模型API调用成本高，需要平衡质量和成本

### Q5：介绍一下你的AI求职助手项目？

**参考答案**：

我独立开发了一个**完整的AI求职助手Web应用**，从产品设计到技术实现全程独立完成。项目规模：**91个代码文件、14,869行代码、23个功能模块、25个页面**。

**核心架构**分为四大板块：

**1. 求职核心链路（4个模块）**：
- **简历诊断**：上传简历后AI生成深度分析报告
- **JD匹配V2**：5维评分（技能/经验/学历/行业/综合）+ 差距弥合方案 + 隐藏需求分析 + 关键词智能分类
- **面试模拟**：AI面试官多轮对话 + 实时反馈
- **AI测评**：5维度AI能力评估，生成个性化学习路径

**2. AI工程层（3个模块）**：
- **模型路由器**：按任务复杂度自动分配lite/standard/pro三级模型，AI成本降低40-60%
- **Prompt优化器**：Few-shot + Chain of Thought提示词工程，提升AI输出质量
- **智能缓存**：模块级TTL策略，最大1000条缓存，减少重复API调用

**3. 增长引擎（3个模块）**：
- **推荐裂变系统**：双向奖励机制——推荐人获100积分+1次面试+1次简历，被推荐人获50积分+1次面试+9折优惠
- **A/B测试框架**：定价策略实验 + 转化率实时仪表盘，数据驱动付费转化优化
- **转化漏斗**：测评→注册→付费全链路追踪，精准定位流失环节

**4. 留存体系（3个模块）**：
- **每日签到 + 连续天数**：建立用户使用习惯
- **成就系统**：解锁里程碑徽章，给予成长反馈
- **积分体系**：积分兑换付费功能，形成留存闭环

**技术栈**：Next.js 14 + TypeScript + Tailwind CSS + MiMo API + Supabase + Creem支付 + Sentry监控

**部署**：阿里云ECS（114.55.110.19）+ Cloudflare Pages双部署

**关键设计决策**：
- 选择模型路由而非单一模型——因为求职场景中，简单的关键词提取用lite模型即可，而深度的JD分析需要pro模型，三级路由在质量和成本之间取得最优平衡
- 选择A/B测试框架——因为定价策略不能靠"拍脑袋"，需要用数据验证哪种价格转化率最高
- 选择游戏化留存——因为求职工具有"用完即走"的天然属性，需要通过签到、成就、积分建立使用习惯

### Q6：你怎么看AI Agent在求职领域的应用？

**参考答案**：
AI Agent的核心价值是**让复杂任务自动化**。以Coze的工作流为例，一个JD诊断可以拆解为：提取关键词→分类技能→匹配分析→生成建议，每个步骤是一个Agent节点。我的JD匹配V2就是借鉴了这个思想，实现了多步骤的自动化分析流程。

### 5.2 面试准备重点

基于我的项目经历，面试中需要重点准备以下三个方向：

#### 🔥 增长策略（Growth Strategy）

**准备要点**：
- **推荐裂变系统**：能说清楚双向奖励的设计逻辑（推荐人100积分+1面试+1简历 vs 被推荐人50积分+1面试+9折），为什么这样设计激励不对称
- **A/B测试**：能举例说明定价策略测试的实验设计（假设：¥29 vs ¥49 vs ¥99的价格点，如何分组、如何判断显著性）
- **转化漏斗**：能说清楚测评→注册→付费各环节的预期转化率和优化策略
- **面试话术**："增长不只是拉新，更重要的是建立可测量、可优化的增长飞轮——我用推荐裂变解决拉新，用A/B测试优化转化，用漏斗追踪定位流失"

#### 🤖 AI工程（AI Engineering）

**准备要点**：
- **模型路由**：能说清楚三级路由的设计逻辑——lite模型处理简单任务（关键词提取），standard模型处理中等任务（简历诊断），pro模型处理复杂任务（JD深度分析），如何定义任务复杂度
- **Prompt优化器**：能解释Few-shot和Chain of Thought的原理，以及为什么结合使用效果更好
- **缓存策略**：能说清楚为什么不同模块需要不同TTL（如职位数据库缓存24h，而JD分析缓存1h）
- **面试话术**："AI产品经理不仅要会用AI，还要会优化AI——模型路由降成本、Prompt优化提质量、缓存策略提速度，这三板斧缺一不可"

#### 🎮 留存设计（Retention Design）

**准备要点**：
- **积分体系**：能说清楚积分的获取路径（签到、完成测评、推荐好友）和消耗路径（兑换面试次数、简历诊断、付费折扣），以及如何平衡积分发放和消耗
- **成就系统**：能举例说明里程碑设计（如"首次完成JD匹配""连续签到7天""推荐3位好友"），以及成就系统对留存的心理学原理（蔡格尼克效应）
- **连续签到**：能说清楚签到奖励的递增设计（第1天10分，第7天50分），以及为什么断签重置能提升次日留存
- **面试话术**："求职工具有'用完即走'的天然属性，我的解法是游戏化——签到建立习惯，成就给予反馈，积分形成闭环。让用户不是'被迫求职'，而是'主动成长'"
