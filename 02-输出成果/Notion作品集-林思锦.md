**时间**：2026年3月至今 | **角色**：产品经理 + 全栈开发 | **技术栈**：Next.js 14 + Tailwind CSS + MiMo API

🔗 **在线演示**：[阿里云ECS](http://114.55.110.19) | [Cloudflare Pages](https://07bc462b.ai-career-tool.pages.dev)

**项目背景**：
2026年95%的应届生在求职中使用AI工具，但市面上的产品大多停留在"通用型AI建议"层面。我独立开发了一款AI求职工具，帮助大学生用AI搞定求职全流程。

**核心功能**：

| 功能 | 说明 | 我做了什么 |
|------|------|-----------|
| 简历诊断 | 上传PDF→5维度评分+优化建议 | 设计评分体系 + 编写Prompt模板 |
| 模拟面试 | 输入岗位→生成题→AI评估 | 设计对话流程 + 面试题库 |
| JD匹配 | 岗位匹配分析 | 设计匹配逻辑 |
| 职业规划 | 输入专业→3条职业路径 | 设计推理Prompt |
| 求职信生成 | AI生成定制求职信 | 设计模板引擎 |
| 面试题库 | 150+题、18家公司 | 数据采集+结构化 |

**项目数据**：
- 代码量：4175行 / 66个文件
- 月运营成本：¥5-10（LLM API调用费）
- 部署：阿里云ECS（生产模式） + Cloudflare Pages（备用）
- 定价体系：免费 + 单次¥9.9-19.9 + 冲刺卡 + 终身¥128

**我学到了什么**：
1. **需求分析**：通过用户调研识别"简历一投多用导致匹配度低"和"面试缺乏练习对象"两大核心痛点
2. **竞品分析**：深度调研5款AI求职工具，识别3个差异化机会点
3. **AI技术理解**：深入理解LLM API调用、Prompt工程、PDF解析等技术
4. **产品设计**：从0到1设计6个功能模块的交互流程
5. **商业化思维**：设计了分层定价体系，理解了"免费获客→付费转化"的逻辑

**时间**：2026年6月 | **角色**：产品方案设计

**项目背景**：

**核心产出**：

- ✅ 3个可交互HTML原型页面（输入页、结果页、加载页）
- ✅ 2个完整Prompt模板（关键词提取、匹配度分析）
