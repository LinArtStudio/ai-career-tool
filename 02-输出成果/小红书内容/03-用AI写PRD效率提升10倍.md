## 🔧 具体方法

### 第1步：需求分析（5分钟）

**Prompt**：
```
请分析以下用户反馈，识别核心需求和痛点：
[粘贴用户反馈]

输出格式：
1. 核心需求（按优先级排序）
2. 用户痛点
3. 解决方案建议
```

**效果**：
- AI自动分析用户反馈
- 识别核心需求和痛点
- 生成解决方案建议

### 第2步：功能设计（10分钟）

**Prompt**：
```
基于以下需求，设计产品功能：
需求：[粘贴需求]

请输出：
1. 功能清单（按优先级排序）
2. 用户故事
3. 交互流程
4. 验收标准
```

**效果**：
- AI生成功能清单
- 自动生成用户故事
- 设计交互流程

### 第3步：文档撰写（10分钟）

**Prompt**：
```
请根据以下信息，生成完整的PRD文档：
- 产品名称：[产品名]
- 目标用户：[用户画像]
- 核心功能：[功能列表]
- 技术方案：[技术选型]

输出格式：
1. 产品概述
2. 用户画像
3. 功能需求
4. 非功能需求
5. 技术方案
6. 项目计划
```

**效果**：
- AI生成完整PRD框架
- 自动补充细节
- 生成技术方案

### 第4步：评审优化（5分钟）

**Prompt**：
```
请评审以下PRD文档，找出潜在问题：
[粘贴PRD内容]

请检查：
1. 逻辑漏洞
2. 遗漏需求
3. 技术可行性
4. 优先级合理性
```

**效果**：
- AI检查逻辑漏洞
- 识别遗漏需求
- 评估技术可行性

## 💡 关键技巧

### 技巧1：给AI清晰的上下文

**错误**：
```
帮我写一个PRD
```

**正确**：
```
请为一个AI客服产品写PRD，目标用户是电商商家，核心功能是自动回复客户咨询。
```

### 技巧2：分步骤生成

**错误**：
```
一次性生成完整PRD
```

**正确**：
```
先生成需求分析 → 再生成功能设计 → 最后生成PRD文档
```

### 技巧3：人工审核和优化

**AI生成的内容需要人工审核**：
- 检查逻辑是否合理
- 补充遗漏的需求
- 优化表达方式
- 添加具体数据
